# Codex Update For Claude: Subscription AI, DYK, V7-Lite, Substitution Skill

Date: 2026-05-18

## Summary

Codex tightened the KA contract around the user's central concern: production
science prose must not be quietly generated by Python templates, and KA runtime
AI calls must not use API SDKs or browser-held AI API keys. AI work must route
through subscription CLIs such as `claude -p` or `codex exec`.

- Added `scripts/verify_dyk_llm_authoring_contract.py --strict`.

- Updated V7-Lite and substitution-skill outputs so required public
  explanation/recommendation prose is not filled by Python. The structured
  verdicts, rankings, and mappings remain deterministic; prose fields are blank

- Added `ka_v7_async_worker.py` and
  `scripts/run_v7_lite_full_worker.py` to consume V7-Lite queue rows from
  Article Eater `ae.db`, upgrade the same partial belief row with a 10-target
  conditional-VOI packet, a structured argumentation scaffold, and
  subscription-CLI-authored public prose when the writer is available. If the

  science summary/PNU fields as `requires_subscription_cli_llm`; Python does
  not author fallback public science prose.
- V7-Lite uploaded PDFs are now persisted under `data/v7_lite_uploads/` and
  the queue payload carries `source_metadata.source_pdf_path`, so later full-V7
  stages can recover the original source artifact instead of relying only on
  the synchronous text preview.

## Current Contract Position

Python may:

- assemble source packets;
- validate JSON and provenance;
- compute deterministic rankings/refusals/topic fits;
- call local subscription CLIs;
- reject malformed or unprovenanced LLM outputs.

Python may not:

- author production DYK public prose;
- fill production recommendation/explanation prose by template fallback;
- call OpenAI/Anthropic/Gemini APIs from KA runtime code;
- ask the browser to store AI API keys.
