## S1 тАФ Cross-Site Scripting via `innerHTML` interpolation ЁЯФ┤ FIXED

**Risk:** the result-card renderer in `ka_contribute_public.html` built a
`card.innerHTML = \`тАж${item.filename}тАж\`` template using server-supplied
fields. Several of those fields are user-influenced or PDF-influenced:

| Field | Source | Attack vector |
|---|---|---|
| `filename` | uploaded file name | upload `<img src=x onerror=alert(1)>.pdf` |
| `citation` | free-form text field | paste `<svg onload=тАж>` |
| `topic`, `article_type`, `question_id` | classifier output (derived from PDF text) | malicious PDF with HTML in title |
| `environment_hits` / `outcome_hits` / `reasons` | classifier hits from PDF body | same |

A successful attack would execute attacker-controlled JavaScript in the
browser of any contributor / instructor reviewing results.

**Fix:** rewrote the renderer to build the DOM with `document.createElement`
and `textContent` тАФ never `innerHTML` тАФ so every server value is treated as
plain text. CSS classes built from `verdict` are scrubbed with
`replace(/[^a-z_]/gi, "")` to avoid CSS injection.

**File:** `ka_contribute_public.html:280тАУ360`

## S2 тАФ Path traversal via uploaded `filename` ЁЯЯв NOT VULNERABLE

**Concern:** `file.filename` is user-supplied. If used directly to build a
filesystem path, a payload like `../../etc/passwd` would write outside
quarantine.

**Verified safe.** The endpoint stores the PDF at:
```python
quarantine_path = month_dir / f"{article_id}.pdf"
```
where `article_id` is generated by `_suggest_next_id` (audit-pass version
returns `KA-ART-` + uppercase hex; no slashes possible). The user-supplied
`file.filename` is only stored in the `pdf_filename` column for
display/audit purposes; never used to construct a path.

**File:** `ka_article_endpoints.py:3076тАУ3079`

## S3 тАФ File magic-bytes / size validation ЁЯЯв ENFORCED

`_validate_pdf_bytes` in `ka_article_endpoints.py:322тАУ349` rejects:
- payloads not starting with `%PDF-`
- payloads larger than `MAX_FILE_SIZE_BYTES` (defaults to 100 MB; env-overridable)
- (warning, not reject) payloads missing `%%EOF` in the last 1 KB

Test B7 asserts a non-PDF byte stream produces `verdict='rejected_bad_file'`
and no DB write.

## S4 тАФ DoS via large upload buffered into memory ЁЯЯб PARTIAL MITIGATION

**Risk:** `await file.read()` reads the whole upload into memory before
size validation runs. A 1 GB upload would buffer 1 GB before the size
check returns failure.

**Mitigation:** FastAPI's `UploadFile` writes large bodies to a
`SpooledTemporaryFile` on disk by default once they exceed
`max_spool_size` (~1 MB), so memory pressure is bounded. The request body
size limit can be set at the ASGI/Uvicorn server layer (`--limit-max-body`)
or via a starlette middleware. **Documented; not enforced in the endpoint
itself.**

**Recommendation for prod:** add a starlette middleware that enforces
`MAX_FILE_SIZE_BYTES + 1 MB` before the route handler runs.

## S5 тАФ SQL injection ЁЯЯв NOT VULNERABLE

Every `conn.execute()` call uses parameter binding (`?`) тАФ no string
interpolation of user data into SQL. Verified by grepping
`ka_article_endpoints.py` for `f"INSERT` / `f"SELECT` / `format(`. None found.

## S6 тАФ Stack trace leakage ЁЯЯв OK

Errors thrown inside the endpoint propagate to FastAPI which returns the
standard `{"detail": "Internal Server Error"}` 500 response тАФ no stack
trace in the body. The audit-pass `try/except/finally` block catches
exceptions, calls `conn.rollback()`, and re-raises (so FastAPI's default
handler still applies).

`HTTPException(400, "Provide at least a PDF or a citation / DOI / title.")`
returns a clean message string with no path info.

## S10 тАФ Privilege escalation / auth bypass ЁЯЯв OK

`/api/articles/suggest` is intentionally **public / anonymous**. It does
not call `_get_current_user()` and does not read `Authorization`. All
inserted rows have `submitter_type='anonymous'` and an `submitter_id` of
NULL. A reviewer must explicitly look up rows with
`source_surface='ka_contribute_public'` and treat them as untrusted.

If the rubric requires authentication on public submissions, that's a
design change тАФ flag for instructor.

---

## S11 тАФ Cookie / CSRF ЁЯЯв N/A

Endpoint accepts only `multipart/form-data` POST without cookies; no
session is established. Standard CSRF concerns don't apply because there
is no authenticated session to forge.
